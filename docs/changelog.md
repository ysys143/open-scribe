## 버전 관리 정책

### 버전 번호 체계
- **MAJOR.MINOR.PATCH**
  - MAJOR: 호환성이 깨지는 변경
  - MINOR: 새로운 기능 추가
  - PATCH: 버그 수정

### 릴리스 주기
- MAJOR: 연 1-2회
- MINOR: 월 1-2회
- PATCH: 필요시 즉시

### 브랜치 전략
- `main`: 안정 버전
- `develop`: 개발 버전
- `feature/*`: 기능 개발
- `bugfix/*`: 버그 수정
- `release/*`: 릴리스 준비
